# gen-art：imagegen 產圖管線

本 skill 的唯一產圖路徑是內建 `image_gen`。**不得使用 Gemini、任何外部 API、`gen_image.py`、`stylize_map.py` 或環境中的 API key 產圖。**

## 何時使用

當 John 要求產生大型美術，例如 NPC／角色／魔物立繪、戰鬥背景、區域地圖、標題圖、圖示或室內背景時使用。16–64px 的像素素材、走路圖、圖磚與敵人戰鬥圖應改用 LPC 或既有 sprite pipeline。

## 執行流程

1. 同時遵守 `imagegen` skill 與專案根目錄 `CLAUDE.md` 的「產圖驗收流程」。
2. **預覽驗收階段**：呼叫內建 `image_gen` 生成候選圖，輸出只可放在 Codex preview／visualization 路徑；不得放入 `godot-project/assets/`、不得更新 CREDITS、不得 import 或測試。
3. John 明確驗收後，才將選定版本複製到專案素材路徑，更新 `CREDITS_素材授權.md` 與相關設計文件，並依改動執行 Godot 檢查。
4. 需要透明背景時，先要求平坦的螢光 chroma-key 底色，再用 `imagegen` skill 指定的去背工具處理；不得改用外部 API。

## 素材類型與構圖

| 類型 | 建議比例 | 用途 |
|---|---:|---|
| 角色／魔物全身立繪 | 3:4 | 設定集、角色頁、任務說明 |
| 對話半身 | 3:4 | 對話框／室內前景 |
| 戰鬥背景 | 16:9 | 側視戰鬥場景，中景留空、無角色 |
| 地區地圖 | 16:9 | 鳥瞰手繪地圖、無文字 |
| 標題圖 | 16:9 | 關鍵美術、無 logo／文字 |
| 圖示 | 1:1 | 主體置中、無文字 |
| 室內背景 | 4:3 | 滿版場景、無角色與文字 |

## 立繪規格

- 人類角色：依 `assets-source/role/ROLE_ART_SPEC.md`，採精緻日系 JRPG 線稿與賽璐璐上色，年齡與服裝忠於設定。
- **魔物立繪**：保留同等完成度與線稿／上色品質，但以獸類、昆蟲、魔物的解剖、姿態、材質與辨識特徵為主；不可套用人臉、人類五官比例或華麗人類服裝。
- 去背圖：背景必須是滿版單一螢光純色，不含漸層、地面、投影、外框或文字；主體沿外輪廓有連續深色描邊。
- 公會任務牆圖：輸出可去背的單色黑墨圖案，不直接烘入羊皮紙、釘子、任務文字或邊框。羊皮紙材質與任務 UI 應在後續組合，才能重複貼到不同紙質上。

## Prompt 基本禁項

- 無文字、字母、數字、logo、浮水印、UI、外框。
- 不加入未指定的角色、背景場景或道具。
- 不在 prompt 放入遊戲名、角色名或任何會被模型畫成文字的內容。
- 不修改 `reference/gdevelop/`。

## 整合後同步

- 新增或替換 `godot-project/assets/` 美術時，更新 `CREDITS_素材授權.md`。
- 新增／移動文件時，更新 `docs/README.md` 或根 README 的對應索引。
- 最終輸出須說明素材用途、存放路徑與驗證結果。
