# Project Instructions

開始任何工作前，必須先完整閱讀專案根目錄的 `CLAUDE.md`，並遵守其中所有規範。

若本文件與 `CLAUDE.md` 有任何衝突，以 `CLAUDE.md` 為準。

## 核心護欄

- 以繁體中文回覆，技術名詞保留英文。
- 只修改任務必要範圍，不擅自變動其他檔案或邏輯。
- `reference/gdevelop/` 是唯讀凍結快照，不得修改。
- 修改遊戲內容、素材、文件或規格後，必須依 `CLAUDE.md` 的「文件同步規則」更新相關文件。
- 修改前先確認 `TASKS/11_並行協作規則.md` 的檔案 owner 與衝突規則。
- 完成前執行與改動相符的 lint、build 與測試；Godot 專案至少執行 headless 檢查。
- 使用 subagent 時，subagent 開工前也必須先完整閱讀 `CLAUDE.md`。
