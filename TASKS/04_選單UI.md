- Task 版本: v1.0
- 狀態: 草稿

## 目標

重建六分頁選單（角色/裝備/道具/地圖/稱號/系統）、HUD（隊伍血條/金幣/目標提示）、商店 UI、對話框 UI，
UI 視覺規格以 John 的 claude.ai/design「Crystal Quest UI」原型（`水晶戰記選單原型.html`／
`水晶戰記戰鬥原型.html`）Tweaks JSON 為定案依據，不是重新設計。

## 產出

- `godot-project/scenes/ui/menu_root.tscn` + 六個分頁子場景/子節點
- `godot-project/scenes/ui/hud.tscn`
- `godot-project/scenes/ui/shop.tscn`
- `godot-project/scenes/ui/dialogue_box.tscn`（MOD-A 直接引用，見 `01_對話劇情.md`）
- UI tokens（顏色/字體/間距）依 DEV 指南 L89：選單 accent 青 `#AADCEB`、戰鬥 accent 金——建議做成
  `godot-project/resources/ui_theme.tres`（Godot Theme 資源），避免各場景各自寫死顏色值。

## 前置依賴

CORE-1、CORE-4（讀 `GameState.party`/`itemInv`/`eqInv`/`gold`）、CORE-6（輸入，含觸控虛擬搖桿/按鈕）。

## 驗收標準

- 六分頁都能開啟、切換、顯示正確資料；血條/數值變化即時反映 `GameState` 異動。
- 觸控與鍵盤兩種輸入都能操作選單（沿用 CORE-6 的統一 action，不另外寫觸控專用分支）。
- 視覺跟 Design 稿 Tweaks JSON 定案參數一致（顏色/字級/版面），有落差要跟 John 確認是否為刻意調整。
