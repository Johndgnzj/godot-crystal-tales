`水晶戰記戰鬥原型.html`）Tweaks JSON 為定案依據，不是重新設計。

### 已知差異／待整合（交協調者）
- **開選單鍵**：build_cq2 用 Esc/M 皆可開選單；本移植收斂為**只用 M 開**（Esc 仍可關選單/商店）。原因：
  Godot 多場景 `_process` 順序不保證，若 Esc 同時能開選單，會發生「Esc 關商店的同一幀誤開選單」競態。
  與 HUD「[M]選單」提示一致。
- **移動鎖定**：world_scene.gd 目前只用 `DialogueSystem.is_busy()` 決定 lock，不知道選單/商店開關。
  選單/商店已提供 `is_open()` 與 `menu_opened/menu_closed`（HUD/選單）signal，整合端（MOD-C/世界場景）
  需把 `menu.is_open()`/`shop.is_open()` 併入移動 lock 條件。本任務未改 world_scene.gd（非擁有檔）。
- **UI 場景尚未掛進世界場景**：hud/menu_root/shop 為獨立場景，未 instance 進 `scenes/world/*.tscn`
  （那些是 MOD-H 擁有檔）。整合端需將三者 instance 進世界場景，並呼叫 `hud.set_scene_id()`/
  `menu.set_scene_id()` 告知目前場景 id（地圖分頁/HudGoal 需要）。
- **eqTitle 型別**：佩戴稱號 id 是 String，`GameState.flag_set` 會 int 化，故稱號分頁與 HUD 直接讀寫
  `GameState.flags["eqTitle"]` 容器（不走 flag_get/set）。建議 CORE-4 檢視是否要提供字串旗標介面。
- **美術紋理**：assets/ui 的 panel/rowhi/bar/face/region_map 等 PNG 尚無 `.import`（無編輯器可產生，
  現有世界場景 .tscn 同樣未引用紋理），故面板底/血條/高亮暫用 ColorRect＋主題色，立繪/區域地圖圖像
  暫略（地圖分頁的位置與等級文字資料完整）。資產匯入後可替換為紋理，版面與資料不需改。
- **音效**：build_cq2 各操作有 sfx（cursor/select/cancel…）；本任務未接音效匯流排（非 MOD-D 範圍），
  待音效系統整合時補呼叫點。
