# MOD-I　美術／音效匯入管線

- Task 版本: v1.0
- 狀態: 草稿
- 對應 GDevelop 系統: `art_v2.py`/`art_v3_lpc.py`/`art_v6_chars.py`/`art_v8_foes.py`/`art_v9_creatures.py`/
  `art_v10_npcwalk.py`/`art_v12_furniture.py`/`art_v13_title.py`/`art_v14_forest.py`（全部是 PIL 圖像合成，
  跟輸出引擎無關，見 `MIGRATION_OVERVIEW.md`「可高度複用」表）

## 目標

把現有美術資產（593 PNG + 17 音效）複製進 Godot 專案並設定正確的 import 設定（像素風格：Filter=Nearest，
不產生 mipmap），美術生成腳本的**合成邏輯保留**，只需要調整輸出路徑，不需要重寫圖像處理演算法。

## 產出

- `godot-project/assets/` 下與 `../gd-crystal-tales/projects/crystal-quest/assets/` 結構一致的複製（
  `char/`/`battle/`/`props/`/`ui/`/`sfx/`/`bgm/`/`map/`），可用同步腳本或一次性複製，複製方式待 CORE-1
  定案（是否用 git submodule/symlink 或直接複製，記錄決策於本檔案）。
- `.import` 設定確認：所有像素美術的 `Filter` 關閉（對應 GDevelop 的 `scaleMode: nearest`），Godot 專案
  設定裡的預設 texture import preset 先調好，避免每張圖手動設定。
- 音效 `mp3`/`wav` 直接複製，Godot 原生支援兩種格式，不需轉檔。
- 室內大圖立繪系統（`intc_<key>` 手繪背景 + `IntArt` 立繪前景，DEV 指南 L88）：Godot 端用
  `TextureRect`/`Sprite2D` 疊圖還原，兩種模式（`mode:"menu"` 立繪選單 / 預設 walk 可走動）都要支援。

## 前置依賴

CORE-1；跟大部分模組平行無依賴衝突（純資產複製與 import 設定），但 MOD-D（室內立繪選單模式的 UI 呈現）
會消費本模組產出。

## 驗收標準

- 抽樣比對複製後的圖檔與原始檔案雜湊一致（不是重新繪製，是原樣複製）。
- 像素圖在 Godot 編輯器內預覽無模糊（Filter 設定正確）。
- `CREDITS_素材授權.md` 的授權義務隨資產複製一併帶到 Godot 專案（複製一份到 `godot-crystal-tales/`，
  之後兩邊素材若各自演進要分別維護，見 `CLAUDE.md` 版本規則）。

## 擁有檔案

`godot-project/assets/**`、Godot 專案的 import 預設設定。

## 已知風險

- `wolf`（野狼）目前是熊剪影去飽和的佔位圖（`CREDITS_素材授權.md` 已標註待補正式素材），遷移時**原樣複製
  佔位圖**，不要在遷移過程中「順便」重繪，正式素材替換是獨立的美術任務，不要跟引擎遷移的驗收標準混在一起。
- 美術生成腳本（`art_v*.py`）本身**不建議在本次任務裡搬進 Godot 專案**：GDevelop 版仍在持續開發（目前到
  第二章），美術生成腳本應該留在 `gd-crystal-tales` 繼續產圖，Godot 專案只消費產出的 PNG，避免兩邊各自
  維護一份生成邏輯導致分岔。若未來要讓 Godot 版獨立產生新美術，屬於遷移完成後的後續任務，不在本次範圍。
