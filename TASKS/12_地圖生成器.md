# MOD-H2　地圖生成器：引擎內 GDScript ＋ typed recipe

- Task 版本: v0.1
- 狀態: 已驗收（生成器完成＋east_forest 示範地區已整合；2026-07-16 實機驗證：生成/smoke/整合驗證全過。唯 Town 實體入口待定，見文末）
- 對應 GDevelop 系統: 承接 MOD-H（`08_地圖管線.md`）；把「新地區生成」從 Python 移到引擎內
- 前置：MOD-H 已產出 5 張可玩 `.tscn`（town/forest/forest2/mine/cave）與 world_scene.gd 契約

## 目標

把「程序生成新地區」的管線從 Python（`gen_maps.py` / `region_gen.py`，字串拼 `.tscn`）改成
**引擎內 GDScript 生成器 ＋ typed recipe（`.tres`）**，對齊業界：**成品地圖＝authored static
scene（在編輯器維護）**、**程序生成只服務新內容**。

## 為什麼（決策紀錄）

- MOD-H 的 `gen_maps.py` 是一次性遷移腳本，靠 `seed=51` 逐格對齊 GDevelop——GDevelop 臍帶已於
  2026-07-14 切斷，此 parity 是歷史包袱；且 Python `random` 與 Godot RNG 不相容，parity 無法跨語言保留。
- 調查發現現有 5 張圖**大多是手工內容**（town 建築/NPC/門/劇情觸發全寫死；鏡草×3、礦坑遺物是綁舊
  迷宮布局的任務道具），且 5 張 `.tscn` 已可獨立運作。→ **重生它們會弄丟任務內容、把 authored 內容
  硬塞 schema，對「乾淨架構」無益**。故決定：**凍結 5 圖為 authored 內容，生成器只服務新地區**。
- （否決方案：「凍結 town、4 地城重生」「5 圖全重生」——回歸風險＋schema 硬塞。）

## 產出

1. **Typed recipe**（`scripts/content/`）：`RegionDef` / `MapDef` / `MapExitDef`（`class_name` +
   `@export` + `from_dict`，對齊現有 `*_def.gd` 慣例）。recipe 存 `resources/map/regions/<id>.tres`。
   **RegionDef 是生成器 dev-time 輸入，不進 `content_db.tres`**（非 runtime 資料）。
2. **GDScript map-kit**（`scripts/map/map_kit.gd`）：`MapGrid`（對應舊 `MapB`）＋靜態演算法
   （`carve_maze` / open layout / autotile / wall_caps / merge_block_rects / reachability BFS /
   邊緣鑿口 / 最遠格 / 選怪），無副作用。
3. **引擎內生成器**（`scripts/map/region_generator.gd`，`extends SceneTree`，headless `-s`）：
   讀 RegionDef → 跑 map-kit → `PackedScene.pack` 產 `.tscn`（滿足 world_scene.gd 17 個 export）→
   `ResourceSaver.save` 產 encounter `.tres`（仿 `build_tres.gd`，不字串拼）→ 印出要接的線
   （SceneRouter.SCENE_PATHS / content_db encounters 聚合 / pacing）。
4. 更新 `gen-region` skill（SKILL.md / test.sh）指向新生成器；移除 `gen_maps.py` / `region_gen.py`。

## 產出手法（沿用既有範本）

- `.tres`：`XxxDef.new()` → `ResourceSaver.save()`（範本 `scripts/content/build_tres.gd`）。
- `.tscn`：程式建節點樹 → 每個子孫 `owner=root`、Player `add_to_group("player", true)` →
  `PackedScene.new().pack(root)` → `ResourceSaver.save()`（範本 `addons/godot_mcp/commands/scene_commands.gd`）。
- autoload 時序：生成器用 `_initialize()` + `await process_frame` 後才取 `ContentDB`（見 memory）。

5. `python3 tests/run_all_tests.py` → 全綠。
