### CH1-D1　第一章 flag / step 藍圖（線性化）
- **問題**：現行用 `step`(序章) + `ch1`(哥布林) + `ch2`(礦山) 三段旗標，對應舊三段式結構。六小節要重編成單一第一章的線性推進。
- **建議**：用單一線性 `ch1_step`（0→13 對應六小節內各節點），保留 `reg`(登錄)、支線旗標(`herb`/`relic`)。舊 `step/ch1/ch2` 全數退場，過場/閘門/遭遇觸發統一改讀 `ch1_step`。
- **✅ 定案（2026-07-19）**：採線性化 `ch1_step`（0→13）。**完整序列見附錄 A**——這是 Phase 2/3 的接線藍圖。
- **產出**：一張「節點→旗標值→觸發什麼(過場/遭遇/閘門)」對照表，寫進本檔附錄，作為 Phase 2/3 的接線依據。
- **影響**：`autoload/game_state.gd`（flag 介面不變，值語意改）、所有 `dialogue/**/*.tres` 的 `when`、場景 `ExitZone.min_step`。

### CH1-D2　礦山守衛角色（小節5）
- **問題**：小節5「想喝酒的礦山守衛」沒有對應 NPC；原 `guard` 已併入羅素隊長，但羅素是南方大道守備隊長（性格正直、地點不符）。
- **建議**：**新增一個輕量礦山守衛 NPC**（暫定 id `warden`），只需一個 `dialogue/npc/warden.tres`（要酒→喝了放行）。成本低，避免扭曲羅素人設。
- **✅ 定案（2026-07-19）**：新增 `warden` 礦山守衛，立繪暫用通用兵。
- **影響**：新增 `dialogue/npc/warden.tres` + 掛進 `dialogue_db.tres`；立繪可暫用通用兵。

### CH1-D3　東之森 ef_* 控制器路線
- **問題**：北礦 `nm_*` 用 `world_scene.gd`（gameplay 掛鉤齊全，補 export 即可）；東森 `ef_*` 用 `painted_screen.gd`（原型控制器，**完全沒有遭遇/NPC/寶箱/過場掛鉤**）。
- **建議**：**把 `ef_*` 遷移到 `world_scene.gd`**（跟 nm_* 統一），一勞永逸，之後所有東森 gameplay 直接受益；不要維護兩套控制器。一次性成本中等。
- **✅ 定案（2026-07-19）**：`ef_*` 統一遷移到 `world_scene.gd`，`painted_screen.gd` 待廢（收尾階段移除）。
- **影響**：9 張 `ef_*.tscn` 的根節點腳本換綁 + 補 export 欄位；`painted_screen.gd` 之後可廢。

## Phase 2 — 主線骨架能打（依 D1、D3；工作量中）

目標：讓 painted 主線**有敵人、有 boss、能推進**。此階段過場可暫用舊素材/占位，重點是「打得到」。

### CH1-04　北礦 nm_* 掛遭遇 + boss
- 做什麼：`nm_a~nm_f` 補 `world_scene.gd` 的 `enc_group`（配 `mine`）。
- **實作細節（2026-07-19 investigation，關鍵）**：光設 `enc_group` **不會觸發遭遇**——`world_scene._is_enc_at()` 查 `enc_rows[ty][tx]=="1"`（TS=tile size），而 nm_* 的 `enc_rows` 目前是**空的**（`_is_enc_at` 恆回 false）。每個 nm_ 場景要**填 `enc_rows`**（走行區鋪 "1"，格數對齊 map_w/map_h 與背景尺寸）。
- boss：`nm_f`(boss房) 的 BossMark **牽涉 ch1_step 條件**（小節3 魔影必敗@step6 vs 小節6 necro@step12 同一深處）→ 條件式觸發歸 **Phase 3 故事 gating**，CH1-04 只做隨機遭遇。
- 依賴：D1。驗收：需**世界場景實例化 harness**（見下 CH1-13 註）headless 確認 `_setup_encounter_tracker` 有建 tracker、`_is_enc_at` 在走行區回 true。

### CH1-05　東森 ef_* 掛遭遇 + boss（含 D3 控制器遷移）
- **D3 遷移配方（2026-07-19 investigation，逐點；nm_a 是對照範本）**：每個 ef_*.tscn——
  1. 根節點 `script` ext_resource：`painted_screen.gd` → `world_scene.gd`。
  2. 根 export 補齊 world_scene 欄位：`map_w/map_h`、`ground_tiles=PackedInt32Array()`(空)、`blk_rows`(空)、`enc_rows`(**填**遭遇地形)、`enc_group`；保留 `scene_id/spawns/camera_zoom`；`default_spawn` world_scene 不用（改由 `GameState.spawn`+spawns）。
  3. **補一個空 `Ground` TileMapLayer 節點**——world_scene L186 `_fill_ground()` 硬存取 `$Ground`，缺則 crash（ground_tiles 空時只是不畫，但節點必須在）。
  4. **移除 `YSort/Player/Sprite` 靜態子節點**——world_scene `_setup_player_visual()` 動態建 AnimatedSprite2D，留著會雙重 sprite。
  5. HUD **不用**加（`_ensure_prompt_ui` runtime 自建）。
- boss：`ef_d`(boss房) 熊複戰(bear_dire) 同樣牽涉 ch1_step（小節5@step11）→ 觸發條件歸 Phase 3。小節1 熊必敗(@step2) 亦 Phase 3 定點戰。CH1-05 只做遷移＋隨機遭遇（`forest`/`forest2`）。
- 依賴：D3、D1。做法：**先遷 ef_a → harness 驗證 loads/_ready 無 crash → 再批次其餘 8**（勿一次改 9 個）。

### CH1-06　場景閘門/推進依新 flag 調整
- 做什麼：Town 東門/北門 `ExitZone.min_step`、各 `cut_on_enter`、`TriggerZone` 的條件全部改讀 D1 的新 flag；`save_manager.gd` 的 `VALID_RESUME_SCENES` 更新為 painted 場景名。
- 驗收：從新遊戲照 flag 序列能一路開放正確的門。

> ▶ **里程碑 M-A**：painted 主線可從芳蕾鎮 → 礦山(序章敗戰) → 回鎮 → 東森 → 礦山(打 necro) 從頭走到尾、每段打得到該打的。過場可還是舊的/簡略。**先驗收這個手感再進 Phase 3。**
