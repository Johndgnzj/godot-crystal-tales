# 世界立繪規格（地圖移動素材；高品質像素・二頭身）

- 版本: v2.0（2026-07-22，依 docs 新架構拆分並自 `世界立繪設計.md` v1.0 改名：本檔只定義「長什麼樣」；去背／交付／現況遷移移至 [`docs/pipeline/世界立繪流程.md`](../pipeline/世界立繪流程.md)）
- 狀態: 草稿（現況 walk＝LPC 像素過渡；目標轉二頭身、優先度最低）
- 適用: overworld **地圖上移動**的素材（走路／待機）。戰鬥素材見 [戰鬥立繪規格.md](戰鬥立繪規格.md)；對話立繪見 [角色立繪規格.md](角色立繪規格.md)；產圖 prompt 見 [`docs/pipeline/prompt/role_world.md`](../pipeline/prompt/role_world.md)、[`enemies_world.md`](../pipeline/prompt/enemies_world.md)

敵人一般不在地圖走動（遭遇即進戰鬥），本檔以我方角色為主；日後若有地圖走動的敵人再比照 `enemies_world`。

---

## 一、動畫與方向

- **walk**：9 幀（第 0＝中性站立，第 1–8＝步態）× **4 方向**。
- overworld **待機**＝ walk 第 0 幀（中性站立）。
- 列序固定 `row0=Up(背面) / row1=Left / row2=Down(正面) / row3=Right`（LPC 標準、對齊 `world_scene.gd`，不可換序）。

## 二、排版與解析度

- 一張 sheet：**列＝方向、欄＝幀**，規則網格、cell 一致、角色畫在格內。
- 建議 cell：二頭身版 **256×256**；LPC 過渡版 64×64（現況）。
- 走路 sheet：9 欄 × 4 列。

## 三、風格

- **目標**：高品質像素風＋二頭身（與 [戰鬥立繪規格.md](戰鬥立繪規格.md) 一致；照角色立繪、縮放 Nearest）。
- **無劍版**：overworld 走路不舉劍（戰鬥才有劍）。
- 現況為 LPC 像素過渡，見 [`docs/pipeline/世界立繪流程.md`](../pipeline/世界立繪流程.md)「現況與遷移」。

## 四、步態（逐幀一致性）

服裝零缺層、腳底基準線一致、root-lock（身體中心 x／腳底錨點逐幀對齊）、只動該動的部位——同 [戰鬥立繪規格.md §四](戰鬥立繪規格.md)。

**walk 第 0 幀＝中性站立**（雙腳併攏，可當戰鬥 idle 站姿基準）。**第 1–8 幀為獨立循環，`frame 8→1` 無縫**，每方向都要看出左右腳完整交替（側面看前後腳交換、正背面看左右腳交換）：

| 幀 | 步態相位 |
|---|---|
| 1 | 左腳接觸：左腳踏出、右腳在後 |
| 2 | 左腳承重：左腳穩定、右腳開始離地 |
| 3 | 左腳 passing：右腳向前通過、雙腳距離收近 |
| 4 | 左腳 recovery：左腳向後收、準備切換右腳 |
| 5 | 右腳接觸：右腳踏出、左腳在後 |
| 6 | 右腳承重：右腳穩定、左腳開始離地 |
| 7 | 右腳 passing：左腳向前通過、雙腳距離收近 |
| 8 | 右腳 recovery：右腳向後收；可無縫接回 frame 1 |
