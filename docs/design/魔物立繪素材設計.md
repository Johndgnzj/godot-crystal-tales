# 魔物立繪素材設計

- 文件版本: v1.2
- 狀態: 定案（礦山、東之森第一批戰鬥圖已整合；全敵人公會懸賞圖已驗收）
- 適用範圍: `godot-project/assets/battle/foe_<enemy_id>_<frame>.png`

本文件定義戰鬥場景的魔物立繪交付規格。數值、掉落與遭遇組合不在此管理，仍以 `resources/content/enemies/*.tres` 與 `encounters/*.tres` 為準。

## 1. 交付位置

每隻魔物先存放於：

```text
assets-source/role/enemies/<enemy_id>/
├── combat_0.png
└── combat_1.png
```

驗收通過後，複製為：

```text
godot-project/assets/battle/foe_<enemy_id>_0.png
godot-project/assets/battle/foe_<enemy_id>_1.png
```

`<enemy_id>` 必須等於敵人資料的 `sprite` 欄位；檔名從 0 起連號。戰鬥載入器會掃描所有連號幀並循環，因此「兩幀呼吸」只保留 `_0`、`_1`，不可遺留舊的 `_2`、`_3`。

## 2. 戰鬥構圖

- 敵方站在畫面左側，角色／英雄在右側；人形與有明確頭部方向的魔物必須**面向右方**。
- 使用透明背景；不可烘入地面、陰影、文字、血條或 UI。
- 輪廓在礦山暗背景上必須清楚，保留深色描邊與乾淨像素邊緣。
- 每隻怪可依體型不同尺寸製作；正式呈現高度由戰鬥場景統一控制，不要為了湊尺寸拉伸原始像素。
- 同一隻怪的所有幀必須有完全相同的畫布大小與水平錨點；垂直起伏必須是有意識的呼吸／飛行動作，不可造成左右漂移或不預期的落地跳動。

## 3. 兩幀呼吸動畫

第一批魔物採最小可用動畫集：`combat_0`（基準）＋`combat_1`（呼吸變化）。戰鬥程式目前每 0.18 秒切換一幀。

| 類型 | `combat_1` 的變化原則 |
|---|---|
| 四足／大型地面怪 | 肩背、胸腔或前爪有可辨識的重量變化；不可只做幾乎看不出的位移。 |
| 人形怪 | 保持面向與武器方向，做小幅重心起伏或手臂／胸腔變化。 |
| 飛行怪 | 本體維持穩定，優先改翅膀角度或振幅；翅膀不可與本體脫離。 |

兩幀要能形成可讀的呼吸節奏，但不得改變怪物的輪廓、朝向、裝備或辨識特徵。若需要攻擊、受傷、死亡等動作，另開動畫集與驗收，不混入呼吸幀。

## 4. 驗收清單

1. 在戰鬥背景上能立即辨識種類與朝向。
2. 循環播放時不露出背景色、不漂移、不混入不同姿勢的舊幀。
3. 飛行怪檢查翅膀與本體是否連續；四足怪檢查重量感是否足夠。
4. 先以預覽圖或 GIF 由設計確認；通過後才覆蓋 `godot-project/assets/battle/`。
5. 整合時更新 `CREDITS_素材授權.md`，並執行 Godot headless 檢查。

## 5. 已驗收範例（2026-07-19）

礦山遭遇怪：`wogol`、`skeleton`、`orc`、`wolf`、`bear`。均採兩幀呼吸；其中骷髏礦工與獸人改為右向戰鬥站姿，洞熊為右向低重心姿勢。

東之森遭遇怪：`bird`、`gslime`、`goblin`、`worm`、`wolf`、`maskedorc`。掠翅鳥採拍翅動作；綠黏史萊姆採壓縮起伏；哥布林拾荒者、巨牙蟲與哥布林頭目採可讀的前壓／重心變化，且哥布林在兩幀間維持持刀手一致。

東之森深處候選怪：`goblin_shaman`、`goblin_tamer`、`wild_hare`、`horn_hare`、`thorn_boar`、`fungus_owl`、`rotwood_beetle`。全部採兩幀動作；野兔與掠角兔共用相同的蓄力跳躍基底，掠角兔僅新增一支短角，供後續進化型沿用。此批已完成戰鬥圖整合，但尚未建立敵人數值資料與遭遇表。

## 6. 設定集與公會懸賞圖（2026-07-19）

大型魔物圖先以原始素材保存，不預先放入未接線的 Godot runtime 資料夾：

```text
assets-source/role/enemies/<enemy_id>/
├── portrait_<enemy_id>.png  # 設定集用；精緻日系 JRPG 全身魔物立繪，透明背景
└── bounty_<enemy_id>.png    # 任務牆用；透明背景的單色粗黑墨圖案
```

- `portrait_*` 保留獸類／魔物本身的解剖、材質與辨識特徵，不套用人類臉型或服飾邏輯。
- `bounty_*` 採簡易、笨拙的中國水墨速寫；只保留少量粗筆與低細節黑墨主體。羊皮紙、文字、釘子與任務框由未來 UI 組合，不烘入圖檔。
- 設定集全身立繪已驗收：`wogol`、`skeleton`、`orc`、`wolf`、`bear`。其中 `wolf` 為跨東之森與北方礦山共用的同一敵人素材。
- 公會懸賞圖已驗收並收錄 22 隻敵人：`goblin_shaman`、`goblin_tamer`、`goblin`、`goblin_chief`、`wild_hare`、`horn_hare`、`thorn_boar`、`fungus_owl`、`rotwood_beetle`、`bird`、`gslime`、`worm`、`wolf`、`bear`、`bear_dire`、`wogol`、`skeleton`、`orc`、`chort`、`necro`、`ogre`、`shadow_demon`。可重現的完整 prompt 見 [`prompt/emenies.md`](prompt/emenies.md) §6。
- 未來圖鑑或公會任務牆 UI 實作時，才依實際用途從本來源複製至 `godot-project/assets/` 並建立引用；目前不改動任何戰鬥、數值或遭遇資料。
