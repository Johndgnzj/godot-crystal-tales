# 魔物立繪 Prompt 集

- 產生工具: OpenAI 內建 imagegen（`gen-art` skill）
- 適用範圍: 設定集魔物全身立繪、任務牆用黑墨懸賞圖
- 相關規格: [角色立繪規格.md](../../design/角色立繪規格.md)「五、敵人立繪」（設定集/懸賞立繪）、戰鬥圖見 [戰鬥立繪規格.md](../../design/戰鬥立繪規格.md)

> 2026-07-19 已產出的東之森深處與礦山圖，產生當下未把 prompt 存檔。以下依當次執行紀錄與已驗收圖重建為**可重出基準**，不是聲稱逐字的歷史原文。從本文件建立起，後續每一隻怪都必須先補齊本檔的個別段落，才可呼叫 imagegen。

## 1. 共同規則

### 1.1 設定集立繪：與角色立繪一致的風格基線

此段必須放在每一隻魔物設定集 prompt 的最前面；唯一可替換的是背景鍵色。

```text
高品質日系動漫 JRPG 魔物立繪，與角色立繪相同的現代日式 RPG／輕小說插畫完成度：乾淨精緻的動漫線稿（線條粗細有變化、非粗黑描邊），細膩的動漫賽璐璐上色搭配柔和漸層與乾淨高光，插畫等級的完成度與豐富細節。主體是魔物，依其獸類、鳥類、昆蟲或異形的合理解剖繪製；不可畫成人類臉型、人類皮膚比例或華麗人類服裝。畫面只有一隻魔物、置中、無外框。
沿魔物外輪廓描一條乾淨連續的深色（#1A1A1A）墨線，約 2-4px、緊貼主體外形，與背景清楚分離、方便去背；這條線只順著主體形狀走，不是在畫面四周加矩形邊界或外框。
背景：單一平塗純色 <鍵色> 滿版填滿整個畫面到邊——無漸層、無材質、無地面、無投影、無邊框、無暗角；背景色不可有光暈打到主體身上，且主體不可出現 <鍵色>。
不要文字、字母、數字、怪物名、logo、浮水印、UI、其他角色或背景場景。
取景：完整單一主體、直幅 3:4，頭頂與足部／翅端都留空，主體置中。
```

- 預設鍵色用螢光綠 `#00FF00`；綠色主體（例如獸人）改用螢光洋紅 `#FF00FF`。
- 產圖後以 chroma-key 去背；成品存為 `assets-source/role/enemies/<id>/portrait_<id>.png`。

### 1.2 公會懸賞黑墨圖：可重複套紙材的透明圖案

此段放在每隻黑墨圖 prompt 的最前面。

```text
公會任務牆用的魔物懸賞圖案：單色純黑墨、粗筆、低細節、易於遠看辨識的全身輪廓。使用粗刷筆與粗鋼筆的輪廓線，只保留少量塊狀內部筆觸；不可細密排線、寫實陰影或精緻插畫渲染。畫面只有一隻魔物、置中、無外框。
背景：單一平塗純色螢光綠 #00FF00 滿版填滿整個畫面到邊——無羊皮紙、無紙紋、無釘子、無地面、無投影、無漸層。主體只能使用黑色墨線，背景不得有其他顏色。
不要文字、字母、數字、怪物名、logo、浮水印、UI、任務框、邊框、其他角色或背景場景。
取景：完整單一主體，置中並保留四周空白。
```

- 產圖後移除綠底，只保留透明背景的黑墨 alpha；成品存為 `assets-source/role/enemies/<id>/bounty_<id>.png`。
- UI 需要羊皮紙時才由 UI 疊底；不可把羊皮紙烘進 `bounty_*`。

### 1.3 每次產圖的紀錄格式

1. 在本檔新增該敵人的「設定集」與「黑墨」兩段完整可複製 prompt。
2. 產出只放 Codex preview；John 驗收前不得放入 `godot-project/assets/`。
3. 驗收後，在段落補上檔名、驗收日期與實際存放路徑；同步更新 `CREDITS_素材授權.md`。
4. 若圖有修正，保留原 prompt，另以「修正版」列出只改動的限制，避免覆蓋歷程。
