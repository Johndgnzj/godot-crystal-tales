## 1. 三塊分工（核心觀念）

地圖製作分三塊，各司其職、**同一個事實只存一處**（這是不脫勾的根本）：

| 塊 | 負責 | 在哪做 | 產物 | 狀態 |
|---|---|---|---|---|
| **A 網頁維護工具** | 拓撲（連通）＋呈現（總覽圖）＋metadata | 本機網頁 | `map-def.json` | ✅ 第一版可用 |
| **B 路徑碰撞** | 可走區 → 碰撞 | Godot `PathPaint32/16` 層刷、Agent 反轉 | 各場景 `.tscn` 的 `CollisionPaint`＋`CollisionDetail` | 🟡 圖層已備、待刷＋反轉（§7）|
| **C 出入口＋骨架** | 建場景骨架、擺 `Area2D`、填目標 | `build_scenes.gd` 依 `map-def.json` 生成 | 各場景 `.tscn`（`Zones`）＋ `SCENE_PATHS` | ✅ 已實作 |

**真相分離原則**：**語意（通到哪、落點）在 `map-def.json`；幾何（碰撞、出入口位置）在 Godot 場景**。兩者用「出口 id」綁定，沒有任何欄位存兩份 → 不會脫勾。

**地格藍圖（terrain）＝貫穿三塊的結構層（選用，見 §2.5）**：在網頁工具對一張圖手畫 40×40 逐格語意（磚路/河/牆/出入口…），存進 `map-def.json` 的 `terrain`。它同時餵三塊——AI 產圖（結構指引）、塊 B（可走格自動代刷 PathPaint）、塊 C（出入口 E 格＝精確開口）。畫了藍圖，三塊都更穩定、更自動；沒畫則三塊維持原行為（向下相容）。

## 2. 資料真相：`map-def.json`

- 位置：`assets-source/map/map-def.json`（單一檔涵蓋所有地區；工具存檔時自動備份 `.json.bak`）。
- 「地圖代碼」與「地區代碼」是 **id，牽一髮動全身（出入口的 `to`、場景檔名都靠它）→ 唯讀**，建錯只能刪掉重建。

### Schema

```jsonc
{
  "version": 1,
  "start": { "region": "M1", "map": "town", "spawn": "home" },  // 新遊戲起點
  "regions": {
    "M2": {
      "name": "北方礦山",          // 地區中文名
      "dir": "north_mine",          // 圖檔資料夾（assets/map/ 下）
      "file_prefix": "nm",          // 圖檔前綴
      "scene_prefix": "NM",         // 場景 id 前綴
      "grid_size": 32,              // 碰撞主格（見下）
      "maps": {
        "a": {
          "kind": "mine",           // town/mine/forest/cave/tower/grassland/other（總覽配色＋主題）
          "floor": 0,               // 樓層，見 §4
          "cell": [0, 1],           // 總覽圖上的 [欄, 列]
          "label": "礦坑入口",       // 顯示地名（選填）
          "boss": true,             // 標記 boss 房（選填）
          "level_band": [2, 5],     // 推薦等級（選填）
          "image": "",              // 圖片 override（選填；空＝用 dir/file_prefix 推導）
          "note": "",               // 設計意圖／任務條件的自由文字（選填；給 agent 讀，非機器判定）
          "terrain": ["....", "..."], // 40×40 地格藍圖（選填，40 個字串×40 字元；見 §2.5）
          "exits": {
            "east": { "side": "east", "to": "b" },
            "north": { "side": "north", "to": "M6", "note": "打完 boss 後才開的捷徑" }
          }
        }
      }
    }
  }
}
```

### 命名規則（推導）

| 東西 | 規則 | 例 |
|---|---|---|
| 場景 id | `scene_prefix` ＋ `KEY`（大寫） | `NM` ＋ `a` → `NMA`；可用 map 的 `scene` 欄位覆寫（如 M1 town → `Town`）|
| 圖檔路徑 | `assets/map/<dir>/<file_prefix>_<key>.png` | `assets/map/north_mine/nm_a.png` |
| 落點 spawn | 自動 `from_<來源圖>`（多口時帶出口 id）| 圖 `a` 進圖 `b` → `b` 有落點 `from_a` |

### 出入口（exit）

`exits` 的 **key ＝ 具名 id**（也是 Godot 端 `Area2D` 的節點名，取看得懂的名字），value：

| 欄位 | 意義 |
|---|---|
| `side` | 在哪一側：`east/west/north/south`（邊緣）、`up/down`（樓梯/地穴）、`interior`（同層內部傳送）|
| `to` | 目標：同區用 map key（`"b"`）、跨區用 `"Mx:key"`（`"M4:a"`）、待接整區用 `"Mx"`（`"M6"`）|
| `require` | 進入條件（選填），如 `"level>=30"`（Godot `ExitZone` 現支援 step 門檻，等級待擴）|
| `note` | 設計意圖／任務條件的自由文字（選填；**給人/agent 讀、非機器判定**——與 `require` 分工：`note` 記「為什麼、什麼任務決定」，`require` 記「程式怎麼判定」。map 節點本身也有同名 `note` 欄）|
| `one_way` | 單向出口（選填）|

**同一側要多個出口**：key 各自取不同具名 id（如 `east_main` / `east_cave`），`side` 都填該側即可。

**連通設計原則（探索感，仿 M2）**：用**迴圈**（cycle）、**三岔口**（3 出口）、**死路藏寶箱**做出探索感；避免整張一路單向直線。房間出口數要有變化（1/2/3 口混用）。

## 2.5 地格藍圖（terrain）— 40×40 逐格語意層

**是什麼**：對一張圖手畫的粗粒度結構藍圖，每格標一種地格語意。它**不是最終畫面**（畫面仍是 AI 手繪 png），而是**貫穿三塊的單一結構真相**，同時餵三條下游（畫了就更穩定、更自動；沒畫則各塊維持原行為）：

| 餵給 | 由誰消費 | 效果 |
|---|---|---|
| AI 產圖 | `blueprint_desc.py` → gen-map-prompt | 藍圖轉中文 `Layout:`/`Exits:`，AI 照結構畫、少畫歪 |
| 塊 B 碰撞 | `blueprint_to_paths.gd` | 可走格自動寫進 `PathPaint32`，免手刷；再照常 `invert_paths.gd` 反轉 |
| 塊 C 出入口 | `build_scenes.gd` | 用「出入口(E)格」把 `Area2D` 擺到精確開口（取代整邊 1120px 粗長條）|

**存哪**：`map-def.json` 的 map 層加選填 `terrain`（與 `cell`/`exits` 同級）＝**40 個字串、每字串 40 字元**（row-major，`terrain[列][欄]`，正好對上 1280÷32＝40×40 格）。未畫藍圖的圖不加此欄。

**地格詞彙（單一定義源）**：`assets-source/map/terrain_palette.json`——網頁塗色盤／`blueprint_desc.py`／`blueprint_to_paths.gd` 三方共讀，改詞彙只改這裡。目前八種：

| 字元 | 語意 | 可走 | 快捷鍵 |
|---|---|---|---|
| `g` 草地 | 明確畫出綠地（與空的差別：空＝透明未定義、草地＝畫出綠地）| ✅ | 1 |
| `B` 磚路　`d` 泥路　`=` 橋　`E` 出入口 | 路面／通道 | ✅ | 2–5 |
| `~` 河　`#` 牆　`^` 山壁　`f` 森林 | 阻擋（森林＝密林樹牆）| ⛔ | 6–9 |
| `.` 空（預設）| 透明留白、未塗＝這個；坑洞或沒畫滿就留這個 | ✅（當開闊地）| `-` |

可走/阻擋由 palette 的 `walkable` 決定：阻擋格（河/牆/山壁/森林）→ 碰撞牆；其餘（含空、草地與未列 code）→ 可走。**空（`.`）＝未定義的留白，排在調色盤最後（快捷鍵 `-`），導出碰撞時視為可走開闊地面；要擋（如坑洞）請畫牆/山壁/河/森林，或日後在 palette 新增專用阻擋 code**。橋(可走)直接覆在河(阻擋)之上、不需特例。空與草地帶 `describe:false`＝**不寫進產圖 Layout**（一般地面靠 gen-map-prompt 的 Theme 帶，避免噪音）。

**怎麼畫**：網頁工具（塊 A）開「🖌️ 藍圖」→ 點一張圖 → 在圖上疊的 40×40 格塗地格 → 存檔（`terrain` 寫進 `map-def.json`）。藍圖可在產圖前先畫（無底圖時鋪灰底），當 AI 的結構指引。
- **工具**：筆刷（拖曳連塗）／填滿（點一下 flood fill 同色連通區）／直線（拖曳 Bresenham）／矩形填滿（拖曳實心矩形）——工具列切換。
- **操作**：左鍵畫・右鍵擦（＝塗回空/留白）・**q/w/e/r 切工具（筆刷/填滿/直線/矩形）**・數字鍵 1–9 選地格、`-` 選空・⌘Z 復原／⌘⇧Z 重做／⌘S 存檔・直線與矩形拖曳時有預覽。
- **清空**：🗑️ 清空鈕重置整張藍圖（可 ⌘Z 復原）。
- **匯入**：📋 匯入鈕從其它地區/地圖複製藍圖（選地區→選圖→預覽→載入），省重畫相似地圖。
- **鄰圖**：🧭 鄰圖預設開——顯示上下左右相鄰地圖（依總覽 cell 相鄰）靠共同邊的一條參考帶，**唯讀、不可畫**，方便把出入口/道路對齊接上鄰圖。

**出入口(E)綁定**：導出時把某邊連續的 E 格對到該邊 `side` 相符的 exit → 精確開口。同一邊多個出口共用時無法唯一對應 → 回退整邊（`build_scenes` 報告的「藍圖出入口」段會提示）。

**畫完的導出指令**（皆綁 `timeout`；順序見 §5）：

```bash
# 塊 C：場景骨架＋精確出入口（讀 map-def.json terrain 的 E 格）
/Applications/Godot.app/Contents/MacOS/Godot --headless -s res://scripts/map/build_scenes.gd --path godot-project
# 塊 B 前置：藍圖可走格 → PathPaint32（[場景名...] 可選，省略＝全部有藍圖的圖）
/Applications/Godot.app/Contents/MacOS/Godot --headless -s res://scripts/map/blueprint_to_paths.gd --path godot-project -- <場景名...>
# 塊 B 收尾：PathPaint 反轉成 CollisionPaint（現成腳本，不變）
/Applications/Godot.app/Contents/MacOS/Godot --headless -s res://scripts/map/invert_paths.gd --path godot-project -- <場景名...>
# 產圖描述（貼進 gen-map-prompt 的 Layout/Exits）
python3 tools/map_editor/blueprint_desc.py <region> <map>
```

## 3. 網頁維護工具（塊 A）

### 啟動

```bash
python3 tools/map_editor/serve.py     # → 瀏覽器開 http://localhost:8770
```

- 純 Python 標準庫，無需安裝套件。
- 用本機 Chrome 開即可（2026-07-21 起 Claude 端 in-app 瀏覽器也連得上 `localhost:8770`，供 agent 驗證用）。
- **縮圖來源＝素材源 `assets-source/`**（不是專案 `godot-project/assets/`）：設計工具以素材源為準，圖片路徑仍是 `assets/map/<dir>/<file_prefix>_<key>.png`，只是 serve 對應到 `assets-source/map/`。素材源與專案已統一為同一套命名（`east_forest/ef_a.png`…），因此 json 一套命名兩邊通用。

### 功能

| 區塊 | 能做 |
|---|---|
| 頂部地區 tab | 切地區；`＋地區` 新增（介面內輸入框，代碼＋名稱）|
| 樓層 bar | 一個地區有多個 `floor` 值時自動出現，切樓層 |
| 工具列 | 連線模式、🖌️藍圖（40×40 地格塗色，見 §2.5）、⇄雙向、🖼️縮圖、🗺️藍圖預覽（總覽節點改顯示地格藍圖＋隱藏連接線＋圖間距歸零貼合，一眼看各圖藍圖接起來是否連得上）、🔍縮放（滾輪以游標為錨／＋−／全覽 fit）、`{}` JSON 檢視器（唯讀）、重載、存檔 |
| 畫布 | 平面網格總覽；跨區目標畫成**地區節點**（虛線方塊，可點跳轉）；中鍵拖曳平移 |
| 地區面板（點空白處）| 編輯名稱／圖檔資料夾／檔名前綴／場景前綴／grid（代碼唯讀）；`＋新增地圖`；`刪除此地區` |
| 地圖面板（點格子）| 大預覽圖；編輯 `image`／label／kind／floor（即時顯示 = 1F/B1）／等級帶／boss／**設計備註 note**；出入口增刪改（side／to／require／**note**／單向）；`刪除此地圖`（代碼唯讀，刪除即時更新）|

- **拖曳排列**：非連線模式下左鍵拖曳格子改 `cell`——拖曳時浮現對齊網格，放到空格＝移動、放到有圖的格＝兩者 `cell` 互換。新增地圖預設接在該區現有群的右側（該區還沒有圖時才用 `(0,0)`）。
- **連線**：開「連線模式」後，每個節點四邊出現 N/S/E/W、中央 U/D/I 的 **side 圓鈕**——點來源 side 鈕定出口方向 → 點終點（可再點終點 side 鈕定回程），side 直接寫入、免事後調；也可直接點兩格自動判方位。開「⇄雙向」一次建來回。
- **藍圖**：開「🖌️ 藍圖」後點一張圖，在其上疊的 40×40 格塗地格；工具有筆刷／填滿／直線／矩形填滿，另有清空、從其它圖匯入、鄰圖對接參考帶（唯讀）；快捷鍵 q/w/e/r 切工具／數字鍵選地格／⌘Z 復原／⌘S 存檔；存檔寫進該圖的 `terrain`。詳見 §2.5。
- **刪除**：二次點擊確認（不走瀏覽器對話框），刪除即時更新畫面。
- **驗證**（底部即時）：缺回程紅線、跨區目標不存在（錯誤）、整區待接／從起點孤島（提醒）。
- **存檔**：寫回 `map-def.json`，覆寫前自動備份 `.json.bak`。

## 4. 樓層（floor）慣例

台灣/日式慣例，**不設 0 樓**（`0` 只留給沒有樓層之分的單層平面地區）：

| 樓層 | `floor` | 工具顯示 |
|---|---|---|
| 地上二樓 | `2` | 2F |
| 地上一樓 | `1` | 1F |
| 地下一樓 | `-1` | B1 |
| 地下二樓 | `-2` | B2 |
| （單層平面地區）| `0` | 地面 |

- **樓層只放 `floor` 欄位，不要塞進地圖代碼（key）**——key 是代碼，會污染場景 id／檔名。
- 一個地區出現 ≥2 種 `floor` 值 → 工具自動切「樓層 tab」，各層各畫一張平面圖。
- 樓層之間的通道靠出入口（樓梯用 `side=up`/`down` 的 exit 互指，開雙向一次建），**與 floor 數值差多少無關**。

## 5. 完整工作流

```
① 在網頁工具編連通/metadata（＋可選：開「🖌️ 藍圖」畫 40×40 地格，見 §2.5）→ 存回 map-def.json
② 產圖：gen-map-prompt 組 prompt（該圖有藍圖 → 先跑 blueprint_desc.py 產 Layout/Exits）→ 得 png → 整合進 assets 並 reimport
③ 叫 Agent → build_scenes.gd 建/同步場景骨架、擺出入口 Area2D、填目標（塊 C；有藍圖 → E 格精確開口）
④ 可走區：有藍圖 → blueprint_to_paths.gd 自動把可走格寫進 PathPaint32；沒藍圖 → 在 Godot 手刷 PathPaint（32 主格＋16 細部）
⑤ 叫 Agent → invert_paths.gd 把 PathPaint 反轉成 CollisionPaint 碰撞（塊 B）
```

- 碰撞格：**32 主格 ＋ 16 細部**（AI 生成的地圖常有細碎邊角需要 16）。`resources/map/collision_tileset.tres` 目前仍是 38，塊 B 開工時改為 32。
- 地圖 png 尺寸**已統一 1280×1280**（＝32×40＝16×80，兩層碰撞皆整除）；2026-07-21 把先前 18 張 1254 舊圖批次縮放至 1280（原圖備份於各 dir 的 `_backup/orig_1254/`），**38 的存在理由（遷就 1254＝38×33）已消除**，塊 B 直接用 32、不再有邊條當牆的問題。

## 6. 塊 C 自動化：`build_scenes.gd`（場景骨架＋出入口生成）

一支通用 headless 腳本，讀 `map-def.json`，一次生成／更新所有「有專案圖、非受保護」的 region×map 場景骨架與出入口。取代舊的 per-地區腳本（`build_painted_region` / `build_north_mine` / `build_floret_town`，已標註停用）。

### 跑法

```bash
# 前置：若素材源 assets-source/map 有新圖或改尺寸，先同步進 godot-project/assets/map 再 reimport
/Applications/Godot.app/Contents/MacOS/Godot --headless --import --path godot-project

# 生成／更新場景（會印報告：生成、保留、碰撞清除、待接 placeholder、缺圖、需登錄 SCENE_PATHS）
/Applications/Godot.app/Contents/MacOS/Godot --headless -s res://scripts/map/build_scenes.gd --path godot-project

# 只同步一張新圖（Region:map），不碰其他場景
/Applications/Godot.app/Contents/MacOS/Godot --headless -s res://scripts/map/build_scenes.gd --path godot-project -- M5:a

# 驗證（載入每張、比對出入口與 map-def、從 Town BFS 連通）
/Applications/Godot.app/Contents/MacOS/Godot --headless -s res://tests/verify_map_scenes.gd --path godot-project
```

### 產出與規則

- **場景檔**：`scenes/world/painted/<file_prefix>_<key>.tscn`，沿用 `ef_a.tscn` 骨架（`world_scene.gd`：Background 圖／空 Ground／空 CollisionPaint／Zones＋出入口／YSort＋Player＋Camera）。SIZE＝1280、`map_w/h=40`。
- **出入口**：`Zones` 下每個 `Area2D`，節點名＝map-def 的 exit key，`side` 決定出口帶位置，`spawn_id`＝同區 `from_<來源key>`／跨區 `from_<來源場景id>`（或沿用既有 legacy 名如 `fromForest`）。落點在「來源出口 side 的相對邊」自動生成。
- **待接**（裸 `Mx` 或目標圖未就緒）：建成 `enabled=false` 的 disabled placeholder（保留場景則不建、僅報告）。
- **CollisionPaint 一律留空**（塊 B 負責）。既有 **38px 舊碰撞會被清除**（塊 B 於 1280 重刷）。
- **不會動的場景（保留）**：① 受保護清單（`Town`：含 NPC／門／過場／實例）；② 既有場景已有 **32px 真實碰撞**（＝塊 B 已完成，如 `nm_*`）；③ 含 NPC／門／Trigger／實例的手工場景。這些的 `enc_group`／`cut_on_enter`／碰撞完全不動；`build_scenes.gd` 用 `SceneState` 讀既有場景保留這些欄位，不硬編碼、不脫勾。
- **新場景要記得登錄 `SCENE_PATHS`**：腳本報告末會列出需加進 `autoload/scene_router.gd` 的條目（跨區出口靠它解析）。

## 6.5 程序生成 layout guide（草稿用，不取代手繪畫面圖）

適用於尚未有手繪背景的新地區。`map-def.json` 仍是連通真相；程序生成器只負責草稿的出口、主路、碰撞與高物件座標。

- 生成的高物件節點都在 `YSort/Prop*`，並標記 group `layout_prop`；手繪整合時保留節點位置與 bottom-center 錨點，只替換其中的 Sprite。
- 用截圖工具的 `scene:<res://...>` 取得含高物件的構圖參考；用 `guide:<res://...>` 隱藏 `layout_prop` 與 Player，取得可供重繪背景的乾淨 guide。
- guide 的草地／路徑／樹只是空間參考，不可直接當正式手繪素材。手繪背景完成後，正式場景仍依本文件的 PathPaint／CollisionPaint 流程處理碰撞。

## 7. 塊 B：路徑碰撞（PathPaint → 反轉 CollisionPaint）

要做碰撞的 30 張場景（ef×9、efd×15、nm×6；Town 另計）已備好四個圖層＋兩套 tileset：

| 圖層 | tileset | 你做什麼 | 執行期 |
|---|---|---|
| `PathPaint32` | `path_tileset_32.tres`（綠格、**無物理**）| 用筆刷把**大面積可走區**刷綠（32 主格，快）| 自動隱藏（`world_scene._hide_paint_layers`）|
| `PathPaint16` | `path_tileset_16.tres`（綠格、無物理）| 邊角細節補刷（16 細格）| 自動隱藏 |
| `CollisionPaint` | `collision_tileset_32.tres`（紅格、有物理）| **不要碰**——反轉器輸出 | 隱藏（`visible=false`）|
| `CollisionDetail` | `collision_tileset_16.tres`（紅格、有物理）| **不要碰**——反轉器輸出 | 隱藏 |

**刷法**：Godot 編輯器打開場景 → 場景樹點 `PathPaint32` → TileMap 面板選那顆綠格 → 在可走處刷；細節換 `PathPaint16`。綠格＝可走，沒刷到＝反轉後會變牆。路徑格「無物理」，刷了不會自己變碰撞（純設計標記）。

**（有藍圖免手刷）**：該圖若已畫 40×40 地格藍圖（§2.5），直接跑 `blueprint_to_paths.gd` 把可走格自動寫進 `PathPaint32`（阻擋格＝河/牆/山壁留空），跳過手刷；之後照樣交給反轉器。仍可事後在編輯器補刷 `PathPaint16` 細節再反轉。

**兩條規則（重要）**：① 可走區要**刷到每個出口所在的邊緣**（玩家才走得到出口 Area2D）；② 出生點那幾格也要可走（反轉器會自動保底落點／出口帶，但順手刷到最保險）。

**反轉（塊 B 收尾，Agent 做）**：刷完 → 交給我 → 反轉器把地圖範圍（32＝40×40、16＝80×80）內「PathPaint 沒刷到」的格子填進 `CollisionPaint`(32)＋`CollisionDetail`(16) 當牆，並保底落點／出口可走。反轉器待第一批刷好後撰寫＋驗證。

素材：`_path_cell.png`（綠標記）＋ `path_tileset_32/16.tres`（無 physics_layer）；`collision_tileset_32/16.tres`（有 physics_layer）為反轉輸出用。舊 `collision_tileset.tres`(38) 已不用。
