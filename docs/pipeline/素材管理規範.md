# 素材管理規範（Asset SOP）

- 狀態: 定案（把既有慣例正式化）
- 最後更新: 2026-07-17

> 解決兩件事：**① John 找/產的素材該放哪　② 素材被拿去用、進了 Godot 專案後怎麼處理**。以下規範是把專案目前**已在跑、但沒寫下來的慣例**正式化（如地圖原圖留 `assets-source/`、授權記 `CREDITS`），不是另立新制。

## 一、素材的三個位置（各司其職）

| 位置 | 角色 | 放什麼 | 進 git? |
|---|---|---|---|
| **`assets-source/`** | **原始素材庫**（可回溯的原檔）| John 找來的原檔、AI 產的原圖（原尺寸、含浮水印/洋紅底/多餘邊）、素材定義表（.xlsx）、來源說明 | ✅ 進（可回溯、團隊共享）|
| **`godot-project/assets/`** | **遊戲實際載入**的成品 | 處理後、遊戲真正吃的 PNG/音檔，**每個都配 `.import`** | ✅ 進 |
| **`CREDITS_素材授權.md`**（repo 根）| **授權帳本** | 每個進 `godot-project/assets/` 的素材的來源/作者/授權/用途 | ✅ 進 |
| **`tmp/`** | **產圖 scratch** | gen-art 中間產物、暫存垃圾 | ❌ 不進（已 gitignore）|

> 📁 **關於此目錄**：`assets-source/` 是**永久保留的原始素材庫**（原圖不刪，供日後重製/改風格/換風格化），與處理後的 `godot-project/assets/` 成對——一個是來源、一個是遊戲實際載入的成品。
> （沿革：原名 `temp_assets/`，因「暫存」二字有可丟的誤導性，2026-07-17 正名為 `assets-source/`；`build_painted_region.gd`／`scene_router.gd` 註解、`gen-map-prompt` skill、`CREDITS`、`MAP_ART_SPEC.md` 的引用已同步更新。）

### `assets-source/` 內部慣例（現況）
- 依素材大類分子目錄，目前有：
  - `map/`：手繪地圖原圖（各地區一夾、與專案同命名，如 `east_forest/ef_a.png`；2026-07-21 統一命名，`_overview/`＝總覽圖、`_backup/`＝舊版）＋ `map-def.json` 地圖定義表（`map-def.xlsx` 已退休為私人草稿）。
  - `role/`：角色立繪／sprite 原圖與切圖暫存（`main/`、`npc/`、`enemies/`，及 role_slicer 匯出的 `<id>/`）。
  - `props/`：互動／場景物件原圖；互動物件依 `props/<art_id>/` 收納 design anchor 與各狀態來源圖（流程見 [`world_object_art/`](world_object_art/workflow.md)）。
  - `sound/`：音效原檔（`attack_sound/`、`incoming_sound/`）。
- **這裡只放素材原檔，不放規格文件**：素材「長什麼樣」的規格在 [`docs/design/`](../design/)、產圖流程在 `docs/pipeline/`（`gen-*` skill 的權威來源）。
- 新增其他大類時比照：`assets-source/<類別>/<批次或來源>/`（例：`assets-source/bgm/`、`assets-source/icon/`）。

## 二、素材生命週期（SOP）

```
① 找到/產出 ──▶ ② 進原始素材庫 ──▶ ③ 處理 ──▶ ④ 進 godot assets ──▶ ⑤ 記授權 ──▶ ⑥ 原圖留存
   (John/AI)      assets-source/<類別>/    去背/裁切/     +.import        CREDITS       assets-source 不刪
                  +來源說明              縮放/正規化     (務必 Reimport)              (供回溯/重製)
```

**逐步規則：**

1. **找到 / 產出**：John 找的第三方素材，或 gen-art 產的圖。先確認授權可用（見第三節）。
   - ⚠️ **gen-art `--out` 相對「執行目錄（repo 根 `godot-crystal-tales/`）」，不是 `godot-project/`**（gen-art skill 文件寫「相對 godot-project」是錯的，2026-07-15 實測）：要一步到位落進遊戲專案就寫完整相對路徑 `--out godot-project/assets/<類別>/xxx.png`；只寫 `--out assets/...` 會在 repo 根憑空生出 `assets/`，得再 `mv` 過去。
2. **進原始素材庫**：原檔放 `assets-source/<類別>/`。同批附**來源說明**（誰做的、從哪來的、授權、日期）——可寫在該子夾的小 README 或直接記進 `CREDITS`。
3. **處理**：去背（洋紅色鍵/flood-fill）、裁切至 bbox、縮放、響度正規化等。處理腳本目前多留在 GDevelop 端 `tools/`（依 TASKS/09 決議），Godot 端只消費產出。
4. **進 godot assets**：處理後的成品放 `godot-project/assets/<類別>/`。
   - ⚠️ **務必 Reimport**：全新 PNG 沒有 `.png.import` → runtime `load()` 讀不到；覆蓋既有檔 → `.godot/imported/` 是舊快取會顯示舊圖。編輯器開著通常自動重匯；否則 FileSystem 面板右鍵 Reimport（或關開編輯器）。
5. **記授權**：在 `CREDITS_素材授權.md` 補一條——來源、作者、授權、用途；**AI 產的**標明工具（如 Gemini/OpenAI Imagen）＋提示詞作者（John/協作 Agent）。
6. **原圖留存**：處理後**原圖不要刪**，留在 `assets-source/` 對應位置，供日後重製或改風格（現況 `north_mine/nm_*` 的原圖就留在 `assets-source/map/north_mine/`，與專案同命名）。

## 三、授權標註規則（重要）

- **只對 John 明確說是 Suno／AI 自產、且未取得商用授權的素材**，才加「不得商用」之類警語。
- **已查證可商用的素材一律不加限制**（例：Pixabay Content License＝免費商用、毋須標註；仍列出以示感謝）。
- 對已可商用的素材硬加警語，會讓 `CREDITS` 自相矛盾、文件失準。找 BGM 優先用 Pixabay Music（可商用）取代 Suno，順便解掉授權疑慮。
- 常見授權對照：
  - **CC0**：可自由用、免署名。
  - **CC-BY / CC-BY-SA / GPL**（LPC 素材群）：需**署名作者**；SA 要求衍生以相同授權分享。→ 這些不能移除署名。
  - **Pixabay Content License**：免費商用、毋須標註。
  - **itch.io 個別包**（如 Anokolisa Pixel Crawler）：多為可商用可改、唯**不得把素材本身當最終產品轉賣**——把它用進遊戲 OK。

## 四、檢查清單（每次動素材對照一次）

- [ ] gen-art 產圖的 `--out` 寫了完整相對路徑（落 `godot-project/assets/…` 或 `assets-source/…`），沒在 repo 根憑空生出 `assets/`。
- [ ] 原檔進了 `assets-source/<類別>/`，且原圖沒被刪。
- [ ] 成品進了 `godot-project/assets/<類別>/`，且已 **Reimport**（有 `.import`、遊戲讀得到）。
- [ ] `CREDITS_素材授權.md` 補了對應授權條目（AI 產的標工具＋提示詞作者）。
- [ ] 授權警語只加在 Suno/AI 未授權者，Pixabay 等可商用者不加。
- [ ] scratch 產物只在 `tmp/`（不進 git）。
