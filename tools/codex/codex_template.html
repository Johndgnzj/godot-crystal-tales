水晶戰記 設定集

十五歲的路德為查明恩人亞倫失蹤的真相，與青梅竹馬瑪琳一同踏上冒險。 水晶碎片、封閉二十年的北方礦山、化為骷髏的失蹤礦工——黑影正在甦醒。

2 人隊伍（上限 4） ATB 戰鬥 單一存檔槽 1280×720 · 像素風 · nearest GDScript

總覽

世界觀、章節主線與技術規格。旗標細節見「劇情與對話」，數值規則見「戰鬥公式」。

芳蕾鎮

故事起點的邊境小鎮。設施：冒險者公會（緹娜）、旅店（朵拉）、神殿（希雅修女）、鎮長宅（巴頓）、道具店（吉德）、鐵匠鋪（漢克）、水井旁（老葛雷）、藥草舖（米拉）、南門（羅素隊長）。鎮內無遭遇戰。

蓋婭女神與記號

本地信仰的大地女神。序章礦山事件後，女神在路德身上留下「女神的記號」（劇情道具 goddess_mark），希雅修女能察覺其變化——主線暗線。

北方礦山

二十年前封閉的礦坑，近來夜裡有異聲。序章見習之地；第二章重返時揭露失蹤礦工已化為骷髏，深處藏著異變的源頭。

水晶碎片

蘊含魔力的稀有素材（crystal_shard），與礦山異變直接相關。中後期敵人與頭目掉落，是貫穿全篇的關鍵物。

冒險者公會

於公會登錄後取得 F 級「公會登錄證」。委託是章節推進的主要載體：第一章討伐哥布林頭目（200G）、第二章受老葛雷私託調查礦工失蹤。

南方大道（伏筆）

通往大城的道路因魔物騷動封鎖，由羅素隊長把守。第二章結束後鎮長囑咐「把礦山所見告訴大城的人」——第三章的出口。

主角隊伍

衍生數值由公式 F-1 計算（含初始裝備）。屬性點：每級 +3 自由分配；技能點：每級 +1，技能上限 Lv10（每級威力 +15%）。

NPC 圖鑑

全員常駐芳蕾鎮。對話依旗標由上而下擇一觸發（先寫後期條件），完整台詞見「劇情與對話」。

敵人圖鑑

DEF 為物理防禦；敵人魔防以 DEF×0.5 計、閃避以 SPD×1.5 計、會心一律 6.25%（見 F-3～F-5）。 ☠ 為頭目（big）：血條不顯示數字。掉落顯示「最終機率（×倍率）」——最終機率＝道具掉落基數 × 怪物倍率，上限 100%（F-10）。

技能

傷害 =（基值×倍率＋固定值）×技能等級加成 − 敵防×0.6，浮動 0.85–1.00；治療無防禦修正。 基值：str/agi 系吃物攻、int 系吃魔攻。技能不判閃避。

裝備

六個裝備格：武器／防具／護腕／靴子／飾品×2（acc1/acc2）。武器・防具・靴子各分 力量／敏捷／法力三型 × 8 階曲線（1~4 階第一章即販售；5~8 階＝II，ch2 旗標後上架）—— 型別只是分類標籤，不限制誰能裝。稀有度（優良／稀有／史詩）為 UI 色階，不影響數值。 護腕未擴充（維持 4 件）；飾品不分型共 40 件。賣價＝買價 30%。裝備 desc 風味文字目前 UI 尚未顯示（選單只列數值）。 詳細設計見 道具武器設計.md。

道具

戰鬥中可用的道具僅限回復 HP／MP 兩類（F-6）。素材目前用途為變賣與未來煉金／鍛造。 「掉落基數」＝base_drop_rate，每個物品各別設定（0 表示不從戰鬥掉落）； 稀有度不參與掉落計算（引擎只讀 base_drop_rate），只是填值慣例（普通素材 20%／優良 12%／稀有 5%）＋UI 色階， 所以會有「優良但基數 0」的消耗品（如上級藥水）。怪物端 drops[].rate 為加乘倍率，見 F-10。

商店

買賣完成時自動存檔。賣出任何裝備／道具一律以其 sell 價成交。

地圖與地區

節奏參數（pacing）定義每張圖的進場等級→目標等級與預計戰鬥數，EXP 縮放係數由此推得（F-9，現場計算）。 寶箱座標為圖塊座標（tx, ty）。

戰鬥公式

摘錄自 specs/BATTLE_FORMULAS.md v2.0。F-1～F-9 抄錄自 GDevelop build_cq2.py（附行號可回溯）； F-10 為 Godot 端刻意新增的平衡設計（2026-07-15 遊玩回饋調整，非 GDevelop 抄錄）。

F-1　衍生屬性 derive(m)

maxhp  = 30 + STR×8 + 裝備hp          maxmp = 8 + INT×5 + 裝備mp
patk   = 6 + 主屬性×2 + 裝備patk       matk  = round(INT×2) + 裝備matk
pdef   = STR + 裝備pdef                mdef  = round(INT×0.8) + 裝備mdef
dodgeV = round(AGI×1.5) + 裝備dodge    critV = 6.25 + AGI×0.15 + 裝備crit（取一位小數）
spd    = AGI

裝備值＝各部位對應屬性加總。critV 取一位小數採 WORLD 版寫法（兩份 derive 不一致的技術債，Godot 版統一）。

F-2　升級經驗 expNeed(lv)

expNeed(lv) = 10 + round(8 × lv^1.4)

F-3　普攻傷害 phys(att, defn)

base = atk×1.8 − def（下限 1）；防禦指令中 ×0.5
會心率：我方 critV／敵方一律 6.25%；會心 ×1.5
final = max(1, round(base × 隨機(0.85~1.00) × 會心倍率))

F-4　閃避判定 dodge(att, defn)

機率% = clamp(守方dodgeV − 攻方AGI×1.5, 0, 30)

只有普攻判閃避；玩家技能傷害與全體技能一律不判（刻意設計，勿補）。敵方 dodgeV 以 SPD×1.5 計。

F-5　技能傷害／治療

技能等級加成 skPow = 1 + 0.15 × (技能Lv − 1)
基值 skBase：int 系→matk（敵人→atk×0.8）；其他→patk（敵人→atk）
防禦 skDef ：int 系→mdef（敵人→def×0.5）；其他→pdef（敵人→def）
傷害 = max(1, round((skBase×mult + flat) × skPow − skDef×0.6) × 隨機(0.85~1.00))
治療 = round((skBase×mult + flat) × skPow)　→ 補至 maxhp 為止，無防禦修正

技能只吃六成防禦（普攻吃全額）——刻意差異，不是筆誤。

F-6　道具使用（戰鬥）

僅 kind = heal / mp 可在戰鬥中使用：heal 補 HP、mp 補 MP，皆 clamp 至上限，使用後扣庫存 1。

F-7　ATB 蓄力

ATB_K = 1.05（全域速度倍率，定案值）
每幀（僅戰鬥 state == "run"）：atb = min(100, atb + (10 + AGI或SPD) × 1.05 × dt)
初始值：我方 random()×40／敵方 random()×30；行動後歸 0
我方優先：任一英雄蓄滿先開指令選單；同幀只處理第一個達標者

選單／演出等非 run 狀態一律暫停蓄力（等待模式由狀態機天然實現）。

F-8　敵人 AI（foeAct）

依序短路判定（非一次加權抽獎）：
① healer：隊友中第一個 HP<55% 者 → 治療 20~30（寫死常數），有對象就結束回合
② 具名技能（foeSkills）：40% → 隨機一招；單體判閃避、傷害 = phys()×mult；全體不判閃避
③ allAttack：30% → 對全體各打一次標準 phys()，不乘倍率、不判閃避
④ fallback：對隨機一人普攻（判閃避）

三旗標可同時存在。以狂暴洞熊為例：具名技能 40%、全體攻擊 60%×30%＝18%、一般攻擊 42%。

F-9　EXP 縮放

實得 EXP = max(1, round(敵方原始 EXP 總和 × EXPSCALE[地圖]))
EXPSCALE = round(隊伍數 × 升級所需經驗總和 ÷ (預計戰鬥數 × 每場平均EXP), 3)

各地圖係數見「地圖與地區」（依 pacing 現場計算，與 exp_scale.gd 同式）。不在 pacing 表內的事件戰（頭目戰等）係數＝1。

F-10　掉落機率（Godot 端新增，v2.0）

對每個敵人的每筆 drops 元素獨立擲骰：
最終掉率 = clamp( 道具 base_drop_rate × 怪物 drops[].rate, 0, 1 )
基數分級：普通素材 0.20／優良素材 0.12／稀有素材 0.05／0＝不從戰鬥掉落

敵方 rate 語意自 v2.0 起是「加成倍率」，不再是絕對機率。base_drop_rate 是 各物品獨立欄位——上列分級只是填值慣例，稀有度本身不參與計算（引擎只讀 base_drop_rate）；drops 內物品 id 查不到時基數以 1.0 計（防呆，此時倍率＝絕對機率）。伴隨再平衡：一般怪 HP ×約1.7、頭目 ×約1.3；pacing 級距收斂 （透過 F-9 EXPSCALE 自動使升級變緩）；一般素材淨掉率降到 20~30%。

存檔規則

存檔物件欄位：v / scene / x / y / flags / party / eqInv / itemInv / gold / chests / auto，寫入 user://cq_save.json。 自動存檔時機：場景進場、帶存檔動作的對話、撿取、開寶箱、買賣完成。室內存檔存「門口外」座標避免重載卡牆。 「重新開始」清空存檔重建預設值；只有一個存檔槽。

場景轉場（SceneRouter）

邏輯場景名 Title / Town / Forest / Forest2 / Mine / Cave / Battle / Hub / EForest1~3 經 SCENE_PATHS 對照表轉實際路徑。 進場定位：result ∈ {win, flee, story, resume} 且 return_x ≥ 0 → 用 return 座標；否則用 spawn 出生點。場景讀完後自行清空 spawn／result。 戰敗特例：lose 強制回 Town、出生點 shrine（教堂重生），收斂在 battle_result() 內。

對話系統（matchWhen）

條件語法只有三種：always／旗標==數字／旗標>=數字（未定義旗標視為 0）。 每個 NPC 的對話表由上而下取第一個成立的條目——越後期的條件要排越前面。 action 為對話結束後的具名副作用；done 旗標保證贈禮類只發一次。過場（CUTS）另支援 battle / transfer / setstep / party 四欄位。

架構備忘：content_db.tres 內嵌副本

遊戲實際只 load() 聚合檔 content_db.tres，其內容是內嵌副本、 不是連結到 resources/content/<分類>/<id>.tres 個別檔案——只改個別 .tres 而沒同步聚合檔， 改動不會反映在遊戲裡（兩邊都要改）。另外 content.json 是舊匯入種子，未含新增的 92 件裝備與 icon/rarity/attr_type 欄位；重跑 build_tres.gd 全量重建前務必先讀 道具武器設計.md 第六、七節。
