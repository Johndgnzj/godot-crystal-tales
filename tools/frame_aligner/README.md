# Sprite 幀對齊工具（frame_aligner）

把多幀 sprite（戰鬥 idle 呼吸、attack…）逐幀**縮放 → 定位 → 統一裁切 → 去背**，輸出成對齊好、
等尺寸的 `<prefix>_0.png、<prefix>_1.png…`。純前端網頁；`serve.py` 可把結果直接存回 `assets-source/`
（符合 CLAUDE.md「產圖驗收流程」的預覽階段，未整合進遊戲）。

戰鬥動畫尺寸/幀數/面向規格見 [`docs/design/戰鬥立繪規格.md`](../../docs/design/戰鬥立繪規格.md)；
交付命名見 [`docs/pipeline/battle_art/workflow.md`](../../docs/pipeline/battle_art/workflow.md)、敵人帳本見 [`prompts/enemy.md`](../../docs/pipeline/battle_art/prompts/enemy.md)。

## 啟動

```bash
python3 tools/frame_aligner/serve.py            # 起服務並自動開瀏覽器（:8779）
```
或由工具集 hub 一起啟動：`./tools/start.sh` → 左側「幀對齊」。
直接用瀏覽器開 `index.html`（file://）也可，但只能「下載 PNG」，不能「存到 assets-source」。

## 核心概念

- **圖層＝幀**：每個圖層就是動畫的一幀。工具主要在調每個圖層的**大小、位置**，加上共用的**輸出尺寸**與**去背**。
- **輸出尺寸（裁切框）**：所有幀共用同一輸出畫布（預設 384×640），保證每幀等尺寸。改尺寸只改畫布、**不會裁到圖片**。
- **參考線**：可加多條直/橫線，拖曳移動、雙擊移除，幫忙對齊。
- **人物比較框**：固定大小的框，**抓框邊可拖曳移動**（抓框內是移動圖層），大小用欄位或「底部置中」。換 idle↔attack 不同輸出尺寸時**維持同大小**，把每個角色縮放到填滿它 → 各角色比例一致。設定存 localStorage 記住。
- **洋蔥皮**：疊前一幀半透明，幫忙對齊呼吸位移。
- **播放遮罩**：按「播放」時把裁切框外**完全遮住**，看到的就是實際輸出畫面。

## 兩種模式

- **合成**：載入多張圖，**每張＝一幀**。各幀各自調大小、位置（例如分開產好的 idle 幀、或把不同素材各放一幀）。
- **切圖**：處理**一張圖含多幀**。載入後自動加第一幀；按「**增加圖層**」用同一張圖再加一幀，各幀自己縮放/平移去框出圖上不同區域（手動切幀，不必先決定幀數）。

## 操作

| 操作 | 方式 |
|---|---|
| 移動目前幀 | 編輯區拖曳，或方向鍵（`Shift`×10）|
| 縮放目前幀 | 滾輪（對準游標）、縮放滑桿、`×½ −10% +10% ×2`、「填滿框」|
| 置中 | 「置中」按鈕 |
| 去背 | 「四角」自動取底色或「吸管」點背景；調容差；despill 去邊緣溢色 |
| 切換/新增/刪除幀 | 縮圖點選、「增加圖層」（切圖）/載入多張（合成）、「刪除此幀」、「複製目前幀」|
| 輸出 | 「存到 assets-source」（經 serve.py 寫回 `assets-source/<目錄>/`）或「下載 PNG」|

## 去背：flood-fill（重要）

從畫面邊界 flood-fill，**只清掉與邊緣相連的背景**——即使主體膚色接近底色，也**不會挖破主體內部**
（舊版單純色距去背在 ogre 這類紅褐膚色上會把肚子挖洞，本工具已改用 flood-fill 解決）。內部沒和邊界相連的
殘色不會被誤刪，必要時調高容差或用吸管處理。

## 安全與邊界

- `serve.py` 只服務工具目錄靜態檔；`/api/frames-save` 只允許寫進 `assets-source/` 底下（擋路徑穿越、驗證
  檔名與 base64、全有或全無寫入），沿用 `role_slicer` 的防護寫法。hub 端同一端點。
- 輸出先進 `assets-source/`；驗收通過後才由人工複製到 `godot-project/assets/battle/` 並建立引用。
