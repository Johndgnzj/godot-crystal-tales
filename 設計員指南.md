# 設計員指南（不需寫程式）

給負責**角色、劇情、數值、道具、武器、美術、地圖**的設計員。你只在 Godot 編輯器裡改**資料與素材**，
不碰程式碼（`autoload/`、`scripts/` 是工程師/AI 的範圍）。

## 開始前
1. 裝 **Godot 4.7**（穩定版）。
2. 用 Godot 開啟 `godot-crystal-tales/godot-project/`（選 `project.godot`）。
3. 左下「檔案系統」面板即可瀏覽 `resources/`（資料）與 `assets/`（美術）。

---

## 一、改數值資料（角色 / 道具 / 武器 / 技能 / 敵人…）

資料是 Godot 原生 `.tres`，在編輯器點開就能在右側 **Inspector** 填欄位，**不用寫 JSON、不用碰程式**。

| 你要改的東西 | 檔案位置 |
|---|---|
| 隊員（角色數值/成長/起始裝） | `resources/content/party/<id>.tres` |
| 裝備 / 武器 | `resources/content/equipment/<id>.tres` |
| 技能 | `resources/content/skills/<id>.tres` |
| 道具（消耗品/素材） | `resources/content/items/<id>.tres` |
| 敵人 | `resources/content/enemies/<id>.tres` |
| 遭遇（每張地圖的敵人組合） | `resources/content/encounters/<地圖>.tres` |
| 商店 | `resources/content/shops/<id>.tres` |
| 寶箱 | `resources/content/chests/<id>.tres` |
| 全域參數（血量/經驗公式係數、隊伍上限…） | `resources/content/derived.tres`、`pacing.tres` |

**改現有實體**：點開該 `.tres` → Inspector 改值 → Ctrl/Cmd+S 存檔。**立即生效**（下次跑遊戲就是新值）。

**新增一個實體**（例：加一把新武器）：
1. 在 `equipment/` 複製一個現有 `.tres`、改檔名成新 `id`（例 `sword_flame.tres`），點開改欄位（含 `id` 欄要一致）。
2. ⚠️ **關鍵一步**：打開 `resources/content/content_db.tres`，在 Inspector 找到對應陣列（`equipment`），
   把新 `.tres` 拖進去 / 加一個元素指向它。**沒掛進 `content_db.tres` 的新實體，遊戲讀不到。**
   （改現有實體不用這步；只有「新增/刪除」要動 `content_db.tres`。）

**道具/武器的 `rarity`（稀有度）與 `icon`（圖示）怎麼填**：兩欄都選填、留空即安全預設值（`rarity` 預設
`common`、`icon` 預設空字串＝暫無圖示）。稀有度判斷邏輯與現有 39 筆資料的完整對照表，見根目錄
`道具武器設計.md`；圖示命名慣例（`assets/battle/icon_<id>.png`）與目前的美術缺口清單也在同一份文件。

> 為什麼要掛進 `content_db.tres`：遊戲靠這一個聚合檔載入全部資料（匯出後才不會出問題）。工程師也可以幫你
> 重跑 `scripts/content/build_tres.gd` 一次重建，但那會**以 `content.json` 為準覆蓋**，平時新增建議直接掛陣列。

## 二、加美術素材

把圖/音檔放進 `godot-project/assets/` 對應子目錄，Godot 一偵測到會**自動 import**（產生 `.png.import`，不用管它）。

| 類型 | 目錄 | 命名慣例 |
|---|---|---|
| 角色行走圖 | `assets/char/` | `<角色>_<方向>_<格>.png` |
| 頭像 / 立繪 / 介面背景 | `assets/ui/` | `face_<id>.png`、`battlebg_<場景>.png`、`menubg.png` |
| 敵人戰鬥圖 / 道具圖 | `assets/battle/` | `foe_<id>_0.png` 等 |
| 物件 / 室內背景 | `assets/props/` | `int_<房間>.png` 等 |
| 地圖 tile / tileset | `assets/map/` | — |
| 音樂 / 音效 | `assets/bgm/`、`assets/sfx/` | — |

**用 AI 生立繪/背景**：有一個 `gen-art` skill（水彩手繪風，走 Gemini）。跟 AI 說「幫某角色畫立繪」即可，
它會依既有風格生成、放進 `assets/`。細節與風格約定見 `.claude/skills/gen-art/SKILL.md`。像素小圖（行走圖/
圖磚）不走這個，用 LPC 素材合成。

---

## 三、地圖

五張世界地圖是場景檔 `godot-project/scenes/world/{town,forest,forest2,mine,cave}.tscn`。用 Godot 的
**TileMapLayer** 直接刷圖磚。（更順手的「地圖製作 skill」規劃中，之後補。）現在要動地圖先找工程師/AI 協助。

---

## 四、改完怎麼確認沒弄壞

- 最快：在編輯器直接按 ▶ 跑遊戲看。
- 或請工程師/AI 跑冒煙測試（`smoke_test.gd`）確認資料都載得起來。
- `.tres` 存檔後若 Inspector 沒報錯、遊戲跑得動，就 OK。

## 五、請不要碰

- `autoload/`、`scripts/`（程式邏輯）、`project.godot`、`tests/` —— 那是工程師/AI 的範圍。
- `reference/gdevelop/`（舊 GDevelop 專案的凍結快照）—— 唯讀參考，別改。
- 數值請改 `.tres`，**不要**去改 `resources/content/content.json`（那只是舊的匯入種子，不是現在的真相源）。

## 六、多人協作
一次多人開工前，看 `TASKS/11_並行協作規則.md`：每個模組/區域有「檔案 owner」，避免兩個人同時改同一個檔。
美術素材（各自的圖）與各自負責的 `.tres` 之間衝突很少，適合平行分工。
